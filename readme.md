## 1. 目标

设计一个数字 IIR 滤波器，使其幅度响应在离散频率轴上尽可能逼近给定模拟滤波器的幅度响应。

许多音频 EQ 插件都实现了具有模拟匹配特性的数字滤波器设计在线算法，即根据用户给定的参数实时计算数字滤波器系数，使其频率响应尽量接近模拟原型。目前比较成熟的算法和研究有：

Martin Vicanek 提出了一系列闭式公式，通过直接构造数字 biquad 滤波器，使其幅度响应在关键频率处匹配模拟原型。

Udo Massberg 通过修改模拟原型滤波器的参数，使其在经过双线性变换（BLT）之后能够在 Nyquist 频率处保持期望的增益。

Sophocles J. Orfanidis 在数字滤波器设计中引入 Nyquist 点增益这一额外约束，通过增加一个自由度，使设计得到的数字滤波器在 Nyquist 频率处满足指定增益，从而减小双线性变换带来的高频失真。

Darabundit 与 Abel 将双线性变换视为一种特殊的共形映射（conformal map），并在其之前引入额外的复平面映射，对模拟频率轴进行重新参数化，以改善 Nyquist 附近的离散化误差。

上述方法大多属于在线算法，即根据参数通过解析公式直接计算滤波器系数。虽然这些方法能够提供较好的近似，但如果希望获得尽可能精确的幅度匹配，离线优化算法可能是一种更有效的选择。

本文的方法通过离线数值优化算法，在离散频率轴上求解数字 IIR 滤波器对模拟幅度响应的最优逼近。

## 2. 使用到的概念和工具

### 2.1 奈奎斯特“亲和性”

我在实验时发现，同一个模拟滤波器原型（如二阶低通滤波器）在不同的截止频率下的拟合效果不同。具体来说，各种优化算法都更容易拟合高频细节而非低频细节。我将这种特性称为“奈奎斯特‘亲和性’”。

如图 2.1.1 和 2.1.2，各种算法对于不同截止频率、相同 Q 的二阶模拟低通滤波器的拟合效果。你可以发现，对于截止频率为 100 的低通效果器的拟合几乎全军覆没；但是对于截止频率为 22000 的低通滤波器的拟合几乎都还好。

### 2.2 频带变换（这里使用一阶全通映射）

首先定义一个一阶全通：Hapf(z, a) = (a - z) / (1 - az)。对于任意一个数字滤波器 H(z)，若将单位延迟 z 替换为 Hapf(z, a)，即 H(Hapf(z, a))，就可以对 H 的频响进行扭曲。

具体来说，假如 w 是原来的频率轴，映射后：

w' = 2 arctan(((1 + a) / (1 - a)) * tan(w / 2))。

即，原本在 w 的复数响应，经过 H'(z) = H(Hapf(z, a)) 的变换后，被搬运到了 w' 处。如果想变换回来，就使用 a' = -a 即可实现，即 H(z) = H'(Hapf(z, -a))。

图 2.2.1 为截止频率为 100 的数字低通滤波器经过不同系数 a 的全通映射的效果。可以见，经过不同系数 a，原本集中在低频的内容，被搬运到了高频，使得奈奎斯特“亲和性”特性可以被利用。这样就可以改善拟合效果了。

![](https://files.mdnice.com/user/176339/5a32aab6-f947-4c57-9194-170d30623cc7.png)

图 2.2.1 fc = 100，Q = 5 的数字低通滤波器经过不同系数 a 的全通映射

### 2.3 现有的系统辨识/优化器轮子

**Matlab：** invfreqz、prony、stmcb、切比雪夫逼近等轮子，以及谱分解最小二乘方法。它们是通过严谨的数学推导实现的传统系统辨识器，对于比较规则的频响（如 ARMA 生成的阶数相同的模拟幅度响应）能够很好地拟合，但对于不规则的频响（如手工生成的理想矩形窗低通），这类算法往往不能很好地拟合。

**Python：** PyTorch 的各种优化器，如 ADAM、LBFGS 等优化器适合解决非线性问题，尤其适合用于这种 ARMA 模型的拟合。但是这往往涉及到寻找最优解问题，所以实现过程充满玄学。效果通常不输给传统系统辨识器。

既然每种算法都有各自的长处，我的实现方法是：把所有算法都做得最好，对于不同的频响，我们取不同算法中最好的结果。

### 2.4 符号计算器 sympy

对于比较复杂的代数推导，可以使用 sympy 来实现。比如说 4 阶 IIR 转 SVF、4 阶 IIR 转 lattice 拓扑的系数推导等过程，使用 sympy 进行推导又快又好。

抑或是 H(Hapf(z, a)) 之类的保阶变换，哪怕是对增益进行缩放 H = (H - 1) * gain + 1 等操作，都能使用 sympy 来实现。最终结果是，输入原系数 b0、b1、b2…、a1、a2… 以及 a 和 gain，程序能自动返回新系数 b0、b1、b2…、a1、a2… 关于原系数和 a、gain 的闭式表达式。

## 3. 实现过程

这一节不打算深入推导每个系统辨识器的底层数学，而是重点记录实验时尝试了哪些方法，如何调用这些轮子，如何把频带变换接到这些方法前后，如何评价结果并自动选出更好的设计。

整体上，我的实现不是“押宝某一个算法”，而是把多种方法统一放进同一个测试框架里跑，然后用统一指标进行比较。代码主入口就是一个多方法比较函数，它会对同一个目标幅度响应分别调用多种设计器、对每一种方法再做一轮 warp 参数搜索，最后输出误差表格、稳定性信息和频响图。

### 3.1 总体流程

整个离线设计流程可以概括为：

1. 在离散频率轴上构造模拟目标幅度响应。
2. 将目标幅度转成能量响应 R(w) = |H(w)|^2。
3. 对每一种 IIR 设计方法：
   - 先直接设计一次，作为 baseline。
   - 再引入一阶全通 warp 的参数 a，搜索更优结果。
4. 对每个结果统一计算误差、极点半径和稳定性。
5. 输出比较表，并绘制目标与各方法的响应曲线。

### 3.2 目标响应的构造方式

在实现里，我并不是直接从某个现成模拟传函开始固定写死，而是先在离散频率网格上构造一个“要拟合的模拟幅度响应”。

先建立一个对数分布的数字频率网格：

1. 采样率 fs = 48000。
2. 频点数 num_points = 8192。
3. 从一个较低频率 f_min 开始，到 Nyquist 为止。
4. 频率轴采用对数分布而不是线性分布。

然后把数字频率 w 换算成实际频率，再用 s = j * 2 * pi * f 在模拟频率轴上生成目标响应。

也就是说，后面所有设计器接收的都不是复频响，而是目标幅度平方响应 R_target。这是整个实现框架里很重要的一点，因为不同轮子虽然内部形式不同，但都被我统一包装成“输入目标能量响应、输出 IIR 系数”的接口。

### 3.3 匹配模拟幅度响应的数字滤波器设计器

我实际实现并测试了下面几种：

#### 3.3.1 谱分解 IRLS 方法

谱分解 IRLS 做法不是直接调用 Matlab 某个现成黑盒，而是自己先做一个迭代加权最小二乘过程，求出分子分母对应的余弦系数，再分别做谱分解，得到稳定的分子多项式和分母多项式。最后再做一次增益校准。

实现步骤：

1. 先把目标功率响应写成一个有理函数逼近问题。
2. 用 IRLS 反复调整权重。
3. 分别恢复分子、分母多项式。
4. 对分母做反射，稳定化系统。
5. 最后重新校准增益。

这条路线的优点是：结构统一、速度较快，而且对“本来就比较像 ARMA 频响”的目标比较有效。

#### 3.3.2 最小相位恢复 + invfreqz

这一条路线里，我先不直接把幅度响应丢给 invfreqz，而是先根据目标幅度使用倒谱域加因果窗的方法恢复一个最小相位复频响，再把这个复频响交给 invfreqz 去拟合。

实现步骤：

1. 先对目标幅度取对数。
2. 用 cepstrum / 最小相位恢复方式重建一个最小相位频响。
3. 再调用 invfreqz。
4. 对结果做稳定化。
5. 对增益重新标定。

也就是说，这里 invfreqz 不是直接裸用，而是被我放在“最小相位恢复”这个预处理之后。这样做的原因很简单：很多时候目标里只有幅度，没有相位，而 invfreqz 更适合拟合复频响，所以我先给它补一个合理的最小相位版本。

#### 3.3.3 最小相位恢复 + Prony/STMBC

思路和前面的 invfreqz 很像，第一步同样是从目标幅度恢复一个最小相位复频响；不同的是，后续不是直接做频域拟合，而是：

1. 先把恢复出的频响插值到线性频率网格。
2. 构造成共轭对称频谱。
3. IFFT 得到脉冲响应。
4. 截取前一段冲激响应。
5. 再用 prony 或 stmcb 去拟合 ARMA 模型。

其中，prony 是直接拟合，stmcb 是进一步迭代细化，stmcb 还会用 prony 的结果作初始化。

所以从工程角度可以把它理解为：先从“目标幅度”恢复出一个近似的最小相位系统，再把问题转换成“由脉冲响应辨识 IIR”。

这两类方法在某些响应上会比纯频域拟合更稳，但也更依赖最小相位恢复和脉冲响应截断的质量。

#### 3.3.4 Yulewalk + stmcb 精修

这条路线并不是简单地调用 yulewalk 一次就结束，而是做了一层包装。

实现步骤：

1. 对目标幅度的对数形式做轻微平滑。
2. 把它压缩成较少的设计节点。
3. 把幅度模板转成功率模板。
4. 用 yulewalk 先得到一个初始 IIR。
5. 再恢复一个平滑的最小相位频响。
6. 转成脉冲响应。
7. 最后用 stmcb 在这个基础上继续细化。

所以这条路线实际上不是单独的 yulewalk，而是：yulewalk 负责给出一个较好的初值，stmcb 负责后续修正。我不知道 yulewalk 是不是这么用的，我发现高 Q 值的频响如果单纯使用 yulewalk 的效果非常差，最后还是通过 stmcb 补救的。

#### 3.3.5 iirlpnorm

这条路线也是先做了一些预处理再调用 Matlab 轮子，而不是直接裸跑。

实现步骤：

1. 对目标对数幅度做平滑。
2. 将频率点压缩成更少的 knots。
3. 根据低频、高频和斜率构造权重。
4. 尝试先用最小相位恢复 + invfreqz 给出初值。
5. 再把这些东西喂给 iirlpnorm。

所以这里 iirlpnorm 的使用方式可以概括为：用平滑后的目标和人为设计的权重曲线，让它更专注于“更值得拟合”的部分；如果能构造出较好的初值，就带初值运行，否则直接裸跑。同样的，我也不知道 iirlpnorm 是不是这么用的，效果也挺差，我还需要再学习学习。

#### 3.3.6 PyTorch 优化器（Adam + LBFGS）

除了 Matlab 的系统辨识轮子之外，我还尝试了一种完全不同的思路：

不再调用任何现成的 IIR 设计函数，而是直接把 IIR 系数本身当作优化变量，使用 PyTorch 的优化器去拟合目标幅度响应。

这里并没有训练什么神经网络，只是借用了 PyTorch 的两样东西：自动求导、优化器（Adam 和 LBFGS）。

本质上是直接优化 IIR 系数，使得数字滤波器的幅度响应尽可能接近目标响应。

实现步骤如下：

1. 将 IIR 分子系数 b 和分母系数 a（除 a0 外）注册为 PyTorch 可训练参数。
2. 在离散频率网格上计算数字滤波器频率响应。
3. 构造损失函数（主要是 dB 域幅度误差，并加入一些权重）。
4. 使用 Adam 进行短暂的预优化。
5. 再使用 LBFGS 进行两阶段精修。
6. 在整个优化过程中记录最优结果。
7. 最后对系数做归一化并重新校准增益。

与 Matlab 的方法不同，这里没有任何“系统辨识公式”，而是把问题完全交给数值优化器。

##### 3.3.6.1 Adam 预热阶段

优化的第一阶段使用 Adam。Adam 的作用不是追求最终精度，而是先把参数从一个比较粗糙的位置推到一个更容易收敛的区域。IIR 系数空间是一个非常非线性的空间，如果一开始直接使用 LBFGS，很容易陷入奇怪的局部解，或者分母系数直接发散。所以这里先跑几十步 Adam，让频响大致靠近目标，再交给 LBFGS 做后续精修。

##### 3.3.6.2 LBFGS 精修

在 Adam 之后，会运行 LBFGS。LBFGS 主要负责把误差快速下降的同时，仍然保持一定的正则约束，防止系数跑得太远，减弱对初始解的约束。

整个优化过程中都会持续记录当前最优的滤波器，如果后续迭代变差，则不会覆盖之前的最好结果。这种做法本质上是：Adam 负责“找方向”，LBFGS 负责“精修”。

##### 3.3.6.3 评价函数

PyTorch 优化器使用的损失函数并不是简单的均方误差，而是一个组合指标，主要包括：

1. dB 域幅度误差。
2. 视觉区域权重（弱化极低阻带）。
3. 曲线斜率误差。
4. 峰值、低频和高频 anchor 点误差。
5. 轻微的系数正则化。

设计一个合理的评价函数的原因是：如果只用简单的均方误差，优化器往往会去修正一些视觉上不重要的地方，而忽略真正影响频响形状的区域。

在很多测试中，使用 PyTorch 优化器轮子得到的拟合结果，不输给 invfreqz 和谱分解 IRLS，在某些情况下甚至还能再压低一点误差。前提是初值能够被 LBFGS 收敛到一个真正的好解。这说明了 LBFGS 对初始位置极其敏感。如果初始位置不好，优化器很容易收敛到一个不理想的局部解，甚至完全跑偏。而如果初始位置足够接近正确解，LBFGS 往往可以非常稳定地收敛到那个“几乎完美”的解。

因此，这一部分后续的研究重点并不在优化器本身，而在于如何构造更好的初始解。目前这一部分仍然在实验中，还没有稳定的实现。

## 5. 总结

通过以上的算法，可以对具有 ARMA 特性的模拟幅度响应进行比较好的拟合。特别的，对于一些极好的虚拟模拟 EQ 实现（如 Kirchhoff-EQ、Fab-Q 等插件），上述的方法同样可以实现：

1. 频响直接拟合，绕过双线性变换，避免了 Nyquist 扭曲。
2. 对任意非整数阶数的模拟原型进行拟合（应该是 Kirchhoff-EQ 开创的先例）。
3. 超过 Nyquist 范围的截止频率调节。
4. 最小相位特性（零延迟）。
5. 通过滤波器结构的拓扑变换，使用 lattice 结构来降低量化噪声，或使用 svf 提高动态调节时的数值稳定性。

这套离线算法的缺点是：

1. 依靠离线计算和打表，无法精确动态调节滤波器特性。
2. 可能有潜在滤波器稳定性问题。
3. 可能存在浪费阶数的情况（这样会浪费算力）。
4. 非最小相位拟合还需另外设计。
